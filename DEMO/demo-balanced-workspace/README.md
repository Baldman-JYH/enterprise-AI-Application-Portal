# Demo A + Demo C 静态原型

## 目录

- `index.html`：主页面
- `styles.css`：样式文件
- `app.js`：交互脚本

## 当前版本能力

这一版已经升级为对话优先的企业 AI 工作入口 demo，重点不是展示应用列表，而是展示“先说需求，再由内部智能体拆解并执行”的工作方式。

### 已支持的核心能力

- 首屏大对话框入口
- 左侧栏默认收起，可展开 / 可隐藏
- ChatGPT 风格的首页聚焦布局
- 多场景切换
- 简单状态机
- 任务规划 / 待确认 / 执行完成三段式链路
- 关键动作确认、非关键动作自动执行

### 已支持的企业场景

- 预定会议
- 发起采购
- 生成周报
- 查制度
- 差旅预定
- 开通权限

## 使用方式

1. 直接用浏览器打开 `index.html`。
2. 默认进入首页，对话框是主入口，左侧栏默认收起。
3. 可通过首页快捷场景、场景卡片或输入自然语言需求发起任务。
4. 支持 `Ctrl+Enter` / `Cmd+Enter` 快速开始处理。
5. 进入流程后，可继续查看规划、确认关键动作并查看执行结果。

## 推荐演示脚本

### 演示一：首页 AI-native 入口

1. 打开首页。
2. 强调这是“所有内部需求的对话入口”，不是传统门户首页。
3. 演示左侧栏默认弱化，焦点集中在大对话框。
4. 强调系统会自动识别意图、调用内部智能体、串联企业系统。

### 演示二：会议预定闭环

1. 选择“预定会议”或直接输入会议需求。
2. 点击“开始处理”。
3. 展示任务规划页。
4. 进入待确认页，说明关键动作由用户确认。
5. 确认执行，展示执行完成结果页。

### 演示三：制度问答与流程分流

1. 选择“查制度”或输入制度类问题。
2. 展示系统先进入规划，再直接输出结果。
3. 强调知识问答场景不强制进入确认页。

### 演示四：更多企业流程场景

1. 切换到“差旅预定”。
2. 再切换到“开通权限”。
3. 说明平台可承载跨部门流程，而不是单点应用。

## 当前定位

这是第二轮打磨后的高保真静态 demo，重点验证：

- 首页是否真正接近 AI 对话入口

- 对话后是否具备智能体执行与状态流转感
- 关键动作确认与自动执行是否清晰
