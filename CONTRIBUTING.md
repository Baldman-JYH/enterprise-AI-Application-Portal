# Contributing

## 分支约定

- `main`：稳定主线
- `feature/*`：功能开发
- `docs/*`：文档更新
- `demo/*`：原型 / demo 打磨
- `chore/*`：仓库维护、结构调整

## Commit 约定

建议使用以下前缀：

- `feat:` 新功能
- `fix:` 修复
- `docs:` 文档更新
- `demo:` 原型与视觉调整
- `chore:` 工程维护

## 提交要求

- 变更产品方向时，同步更新 `PROJECT_DOCS/` 相关主文档。
- 完成一个阶段性动作后，同步更新 `GPT_doc/企业AI应用门户_进展.md`。
- 如果修改 demo，请同步更新 `DEMO/demo-a-c/README.md`。
- 不要把 `.codex/`、本地编辑器配置和无关临时文件提交进仓库。

## Pull Request 要求

每个 PR 至少说明：

1. 本次目标是什么。
2. 涉及哪些文件或页面。
3. 是否影响 demo 展示或主线文档。
4. 是否完成基本验证。

## Issue 使用建议

- 新需求：使用 `Feature request`
- 缺陷：使用 `Bug report`
- 文档整理：使用 `Documentation task`
